### Component Placement Rules

#### Desktop (1200px+)
- **Container Width**: 1200px max-width, centered
- **Grid**: 12-column grid with 24px gutters
- **Navigation**: Fixed header, 72px height
- **Hero**: 600px minimum height
- **Project Cards**: 3-column grid (350px each)
- **Content**: 2-column layout where appropriate
- **Sidebar**: 300px width for supplementary content

#### Tablet (768px - 1199px)  
- **Container Width**: 100% with 24px padding
- **Grid**: 8-column grid with 20px gutters
- **Navigation**: Condensed navigation, logo + hamburger
- **Hero**: 500px minimum height, stacked CTAs
- **Project Cards**: 2-column grid
- **Content**: Single column with wider text blocks
- **Sidebar**: Stacks below main content

#### Mobile (320px - 767px)
- **Container Width**: 100% with 16px padding
- **Grid**: 4-column grid with 16px gutters
- **Navigation**: Hamburger menu, optimized touch targets
- **Hero**: 400px minimum height, vertical CTA stack
- **Project Cards**: Single column, full width
- **Content**: Single column, increased line-height
- **Typography**: Reduced scale (-10-15%)

### Accessibility Layout Requirements

#### Focus Management
- **Tab Order**: Logical navigation flow (header → hero → projects → skills → footer)
- **Skip Links**: "Skip to main content" for screen readers
- **Focus Indicators**: 2px outline, high contrast color

#### Touch Targets
- **Minimum Size**: 44x44px for all interactive elements
- **Spacing**: 8px minimum between touch targets
- **Gesture Support**: Swipe gestures for project carousel on mobile

#### Screen Reader Optimization
- **Landmark Roles**: Clear section roles (banner, main, complementary, contentinfo)
- **Heading Hierarchy**: Proper H1→H2→H3 structure
- **Alt Text**: Descriptive alternative text for all images
- **Live Regions**: Dynamic content announcements

### Performance Layout Considerations

#### Critical Path Optimization
- **Above-the-fold**: Hero content loads first
- **Progressive Enhancement**: Core functionality works without JS
- **Lazy Loading**: Below-fold images and components

#### Layout Shift Prevention
- **Placeholder Dimensions**: Reserved space for dynamic content
- **Font Loading**: Fallback fonts with similar metrics
- **Image Dimensions**: Width/height attributes prevent reflow

### Interactive Element Specifications

#### Hover States
- **Project Cards**: translateY(-4px) + shadow enhancement
- **Buttons**: 2px elevation + color darken
- **Links**: Underline + color change
- **Navigation**: Background highlight

#### Loading States
- **Skeleton Screens**: Animated placeholders for slow-loading content
- **Progressive Enhancement**: Core content visible immediately
- **Loading Indicators**: Spinner animations for async actions

#### Error States
- **Graceful Degradation**: Fallback content for failed components
- **User Feedback**: Clear error messages with retry options
- **Offline Support**: Cached content when possible

This wireframe specification ensures consistent, accessible, and performant layouts across all device sizes while maintaining the professional appearance required for AI/ML engineer positioning.
